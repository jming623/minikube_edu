# Kubernetes 자주 쓰는 명령어

---

## 클러스터 기본 확인

```bash
minikube status                        # 클러스터 상태 확인
minikube start                         # 클러스터 시작
minikube stop                          # 클러스터 중지

kubectl get nodes                      # 노드 목록
kubectl get nodes -o wide              # 노드 상세 (IP, 역할 등)
kubectl cluster-info                   # API 서버 주소 확인
kubectl config current-context         # 현재 연결된 컨텍스트 확인
```

---

## 리소스 조회

```bash
# 기본 조회
kubectl get pods                       # 현재 네임스페이스 Pod 목록
kubectl get pods -o wide               # Pod 상세 (IP, 노드 등)
kubectl get pods -A                    # 전체 네임스페이스 Pod 목록
kubectl get pods -n <namespace>        # 특정 네임스페이스 Pod 목록
kubectl get pods --show-labels         # 레이블 포함 조회
kubectl get pods -l app=my-nginx       # 특정 레이블 필터링

# 자주 쓰는 리소스
kubectl get all                        # 주요 리소스 한번에 조회
kubectl get deployments
kubectl get services
kubectl get configmap
kubectl get secret
kubectl get pvc
kubectl get ingress

# 네임스페이스
kubectl get namespaces
```

## describe (상세 정보)

리소스의 현재 상태, 이벤트, 설정값을 자세히 볼 때 사용

```bash
kubectl describe pod <POD_NAME>
kubectl describe deployment <DEPLOYMENT_NAME>
kubectl describe service <SERVICE_NAME>
kubectl describe node <NODE_NAME>

# 예시
kubectl describe pod my-nginx-7d8b9c-xxxxx

# 네임스페이스 지정
kubectl describe pod <POD_NAME> -n <namespace>
```

**describe에서 주로 확인하는 항목:**

| 항목 | 위치 |
|------|------|
| Pod가 어느 노드에 떴는지 | `Node:` |
| 컨테이너 상태 / 재시작 횟수 | `State:`, `Restart Count:` |
| 환경변수, 마운트 볼륨 | `Environment:`, `Mounts:` |
| 문제 원인 | `Events:` (맨 아래) |

## 리소스 YAML 파일 보기

실행 중인 리소스의 전체 YAML 스펙을 출력

```bash
kubectl get pod <POD_NAME> -o yaml
kubectl get deployment <DEPLOYMENT_NAME> -o yaml
kubectl get service <SERVICE_NAME> -o yaml
kubectl get node <NODE_NAME> -o yaml

# 파일로 저장
kubectl get deployment my-nginx -o yaml > backup.yaml
```

> `-o json` 으로 바꾸면 JSON 형식으로도 출력 가능

## YAML에서 특정 값만 출력 (jsonpath)

`-o jsonpath` 옵션으로 YAML 구조 안의 원하는 값만 추출

```bash
# Pod의 IP 주소
kubectl get pod <POD_NAME> -o jsonpath='{.status.podIP}'

# 컨테이너 이미지 이름
kubectl get pod <POD_NAME> -o jsonpath='{.spec.containers[0].image}'

# 노드 이름
kubectl get pod <POD_NAME> -o jsonpath='{.spec.nodeName}'

# Service의 NodePort 번호
kubectl get service <SERVICE_NAME> -o jsonpath='{.spec.ports[0].nodePort}'

# 전체 Pod 목록의 이름만 출력
kubectl get pods -o jsonpath='{.items[*].metadata.name}'

# 여러 값을 동시에 출력 (탭 구분)
kubectl get pods -o jsonpath='{range .items[*]}{.metadata.name}{"\t"}{.status.podIP}{"\n"}{end}'
```

**jsonpath 경로 찾는 방법:**
1. 먼저 `-o yaml` 로 전체 구조 확인
2. 원하는 값의 경로를 `.` 으로 연결
3. 배열은 `[0]` 또는 `[*]` (전체) 사용

## 로그 확인

```bash
kubectl logs <POD_NAME>                        # Pod 로그
kubectl logs <POD_NAME> -f                     # 실시간 로그 스트리밍
kubectl logs <POD_NAME> --tail=50              # 최근 50줄만
kubectl logs <POD_NAME> -c <CONTAINER_NAME>    # 멀티 컨테이너 Pod일 경우
kubectl logs <POD_NAME> --previous             # 재시작 전 이전 컨테이너 로그
```

## Pod 접속 및 명령 실행

```bash
# Pod 안으로 shell 접속
kubectl exec -it <POD_NAME> -- /bin/bash
kubectl exec -it <POD_NAME> -- /bin/sh        # bash 없을 때

# 접속 없이 단일 명령 실행
kubectl exec <POD_NAME> -- env                # 환경변수 출력
kubectl exec <POD_NAME> -- cat /etc/nginx/nginx.conf
```

## 배포 및 적용

```bash
kubectl apply -f <FILE>.yaml                  # 생성 또는 수정 적용
kubectl apply -f .                            # 현재 디렉토리 전체 yaml 적용
kubectl delete -f <FILE>.yaml                 # yaml 기반 삭제
kubectl delete pod <POD_NAME>                 # Pod 강제 재생성 (삭제 후 자동 재생성)
```

## 롤아웃 (배포 상태)

```bash
kubectl rollout status deployment/<NAME>      # 배포 진행 상태 확인
kubectl rollout history deployment/<NAME>     # 배포 히스토리
kubectl rollout undo deployment/<NAME>        # 이전 버전으로 롤백
```

---

## 이벤트 확인 (트러블슈팅)

```bash
kubectl get events                            # 현재 네임스페이스 이벤트
kubectl get events --sort-by=.lastTimestamp   # 시간 순 정렬
kubectl get events -n <namespace>
```
