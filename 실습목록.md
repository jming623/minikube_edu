# Kubernetes 심화 실습 목록

> 기반 환경: WSL2 Ubuntu + Docker + Minikube (멀티노드)
> 기본 실습(nginx 배포, NodePort 서비스) 완료 후 진행

---

## 1. YAML 매니페스트 기반 배포 관리

- **목표**: kubectl create 명령어 대신 YAML 파일로 리소스를 선언적으로 관리하는 방법 익히기
- **다루는 내용**:
  - Deployment, Service YAML 작성
  - `kubectl apply -f` 를 통한 선언적 배포
  - Label, Selector, Annotation 활용
  - 리소스 수정 및 재적용 흐름

---

## 2. ConfigMap & Secret

- **목표**: 애플리케이션 설정값과 민감 정보를 Pod 외부에서 주입하는 패턴 익히기
- **다루는 내용**:
  - ConfigMap 생성 (파일, 리터럴)
  - Secret 생성 및 base64 인코딩
  - 환경변수 주입 vs 볼륨 마운트 방식 비교
  - Pod에서 설정값 확인

---

## 3. Rolling Update & Rollback

- **목표**: 무중단 배포와 장애 시 롤백 전략 이해하기
- **다루는 내용**:
  - Deployment의 strategy (RollingUpdate) 설정
  - maxSurge, maxUnavailable 조정
  - 이미지 버전 변경을 통한 롤링 업데이트 실행
  - `kubectl rollout status`, `kubectl rollout undo` 활용
  - Revision 히스토리 확인

---

## 4. Resource Limits & Requests + HPA (오토스케일링)

- **목표**: Pod 리소스 제한 설정과 부하 기반 자동 스케일링 구현하기
- **다루는 내용**:
  - CPU/Memory requests, limits 설정
  - metrics-server 활성화
  - HPA(Horizontal Pod Autoscaler) 생성
  - 부하 테스트로 스케일 아웃/인 확인
  - `kubectl top pods`, `kubectl top nodes`

---

## 5. Namespace를 활용한 환경 분리

- **목표**: dev/staging/prod 등 환경을 Namespace로 논리 분리하는 패턴 익히기
- **다루는 내용**:
  - Namespace 생성 및 리소스 배포
  - Namespace 간 리소스 격리 확인
  - ResourceQuota, LimitRange 적용
  - `kubectl` 기본 Namespace 전환 (context)

---

## 6. Probe (Health Check) 설정

- **목표**: 컨테이너 상태를 자동 감지하고 자가 치유하는 메커니즘 이해하기
- **다루는 내용**:
  - livenessProbe: 컨테이너 재시작 조건
  - readinessProbe: 트래픽 수신 준비 상태
  - startupProbe: 느린 초기화 앱 대응
  - HTTP, TCP, Exec 방식 비교
  - 의도적 장애 주입으로 자가 복구 확인

---

## 7. Ingress를 활용한 HTTP 라우팅

- **목표**: 하나의 진입점에서 여러 서비스로 라우팅하는 패턴 익히기
- **다루는 내용**:
  - minikube addons enable ingress
  - Ingress 리소스 작성 (호스트 기반, 경로 기반)
  - 여러 서비스를 하나의 도메인으로 노출
  - /etc/hosts 편집으로 로컬 도메인 테스트

---

## 8. PV / PVC (퍼시스턴트 볼륨)

- **목표**: Pod가 재시작되어도 데이터가 유지되는 영구 저장소 패턴 익히기
- **다루는 내용**:
  - PersistentVolume, PersistentVolumeClaim 생성
  - StorageClass 개념
  - Pod에 PVC 마운트
  - Pod 삭제 후 데이터 영속성 확인

---

## 9. Job & CronJob (배치 작업)

- **목표**: 일회성/주기적 배치 작업을 Kubernetes에서 실행하는 방법 익히기
- **다루는 내용**:
  - Job 생성 및 완료 상태 확인
  - 병렬 처리 (parallelism, completions)
  - CronJob 스케줄 설정
  - 실패 정책 (backoffLimit, restartPolicy)

---

## 10. Multi-Container Pod 패턴 (Sidecar)

- **목표**: 하나의 Pod에 여러 컨테이너를 구성하는 현업 패턴 익히기
- **다루는 내용**:
  - Sidecar 패턴: 로그 수집기 컨테이너 추가
  - InitContainer: 메인 앱 실행 전 초기화 작업
  - 컨테이너 간 볼륨 공유
  - Ambassador, Adapter 패턴 소개

---

## 11. RBAC (역할 기반 접근 제어)

- **목표**: 클러스터 내 사용자/서비스 권한을 제어하는 방법 이해하기
- **다루는 내용**:
  - ServiceAccount 생성
  - Role, ClusterRole 정의
  - RoleBinding, ClusterRoleBinding
  - 특정 Namespace에서만 동작하는 제한된 권한 테스트

---

## 12. Helm 차트를 활용한 패키지 배포

- **목표**: Kubernetes 애플리케이션을 패키지로 관리하고 재사용하는 방법 익히기
- **다루는 내용**:
  - Helm 설치 및 기본 명령어
  - 공개 차트로 앱 배포 (예: MySQL, Redis)
  - values.yaml 커스터마이징
  - 나만의 Helm 차트 생성 기초
